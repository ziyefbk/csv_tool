# Windows 启动说明

## ⚠️ 重要提示

由于 Tauri 的 `beforeDevCommand` 在 Windows 上可能有问题，我们采用手动启动方式。

## 🚀 推荐启动方式（两步法）

### 步骤 1: 启动前端开发服务器

打开**第一个** PowerShell 窗口，运行：

```powershell
cd frontend
npm run dev
```

等待看到类似输出：
```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
```

**保持这个窗口打开！**

### 步骤 2: 启动 Tauri 应用

打开**第二个** PowerShell 窗口，运行：

```powershell
cd tauri
cargo tauri dev
```

应用窗口会自动打开。

## 🔄 一键启动脚本（实验性）

也可以尝试运行 `运行GUI.bat`，它会自动启动前端服务器和 Tauri。

如果遇到路径问题，请使用上面的两步法。

## 🛑 停止应用

1. 在 Tauri 窗口按 `Ctrl+C` 停止后端
2. 在前端窗口按 `Ctrl+C` 停止前端服务器

## 📝 常见问题

### 问题：端口 5173 被占用

**解决方案：**
1. 查找占用进程：
   ```powershell
   netstat -ano | findstr :5173
   ```
2. 结束进程或修改端口（编辑 `frontend/vite.config.ts`）

### 问题：前端服务器启动失败

**解决方案：**
1. 确保已安装依赖：`cd frontend && npm install`
2. 检查 Node.js 版本：`node --version`（需要 v18+）

### 问题：Tauri 启动失败

**解决方案：**
1. 确保前端服务器正在运行（http://localhost:5173）
2. 检查是否安装了 Tauri CLI：`cargo tauri --version`
3. 如果未安装：`cargo install tauri-cli --version "^1.5" --locked`
