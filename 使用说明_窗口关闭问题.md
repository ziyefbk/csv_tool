# 解决 BAT 文件窗口立即关闭的问题

## 问题

双击 BAT 文件后，窗口立即关闭，看不到任何错误信息。

## 原因

1. 脚本执行出错后立即退出
2. 脚本末尾的 `pause` 命令没有执行
3. 编码问题导致脚本无法正确执行

## 解决方案

### 方法 1: 使用命令行运行（推荐）

**不要双击 BAT 文件**，而是：

1. **打开 PowerShell 或命令提示符**
2. **切换到项目目录**：
   ```powershell
   cd C:\Users\28275\Desktop\csv_tool
   ```
3. **运行脚本**：
   ```powershell
   .\构建EXE.bat
   ```

这样可以看到完整的输出和错误信息。

### 方法 2: 使用调试版脚本

已创建 `构建EXE_调试版.bat`，它会显示更多调试信息。

### 方法 3: 修改脚本末尾

如果脚本末尾没有 `pause`，添加：

```batch
echo.
echo 按任意键退出...
pause >nul
```

### 方法 4: 检查常见错误

窗口立即关闭的常见原因：

1. **缺少 Rust**
   - 错误：未找到 `rustc` 命令
   - 解决：安装 Rust from https://rustup.rs/

2. **缺少 Node.js**
   - 错误：未找到 `node` 命令
   - 解决：安装 Node.js from https://nodejs.org/

3. **路径错误**
   - 错误：找不到 `frontend` 或 `tauri` 目录
   - 解决：确保在项目根目录运行脚本

4. **编码问题**
   - 错误：脚本中的中文字符无法正确显示
   - 解决：使用 PowerShell 运行，或修改脚本编码

## 推荐做法

**始终使用命令行运行 BAT 文件**，而不是双击：

```powershell
# 在 PowerShell 中
cd C:\Users\28275\Desktop\csv_tool
.\构建EXE.bat
```

这样可以：
- ✅ 看到完整的输出
- ✅ 看到错误信息
- ✅ 窗口不会立即关闭
- ✅ 方便调试问题

## 快速测试

运行以下命令检查环境：

```powershell
# 检查 Rust
rustc --version

# 检查 Node.js
node --version

# 检查 Tauri CLI
cargo tauri --version

# 检查目录
Test-Path frontend
Test-Path tauri
```

如果所有检查都通过，再运行构建脚本。
